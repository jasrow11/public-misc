Methodology, scoring & tier definitions

How to read the matrix

Two independent dimensions per cell. Support (Yes / Partial / No / N/A) is what the platform can do; Confidence (V / L / U) is how sure the assessment is, and is orthogonal to support. The IDS and IPS columns use a delivery scale instead: Built-in (native, marked N) / Add-on (separate module, container, or premium tier, marked A) / None. Sort any column by clicking its header (click again to reverse); the Product and Score headers sort alphabetically and by score.

Score

Weighted feature count: each Yes = 1 point, each Partial = 0.5, No and N/A = 0, summed across all 63 feature columns and rounded to one decimal. Products are sorted by this score, highest first. The score is absolute (not normalized), so platforms with many N/A cells - notably cloud-native services whose NAT/ALG functions live in separate managed products - naturally score lower because they support fewer of these specific features.

Scoring thresholds by feature group

Rule & Policy Management. Yes = first-class per-rule counters/grouping exposed in the product's primary interface (GUI or well-supported CLI). Partial = achievable but requires the underlying engine/CLI, a plugin, or a workaround, or is limited in scope. No = not supported. N/A = concept does not apply.
NAT / CGNAT. Yes = natively supported and documented. Partial = via add-on/plugin, external module, or limited/experimental support. No = not supported. N/A = the product does not perform this function because it is delivered by a separate platform service (e.g. a cloud NAT gateway). CLI-driven support on a network OS counts as Yes. IP aliasing = more than one public IP bound to a single WAN physical interface (secondary addresses / virtual IPs); on host-based packages this is an OS-level capability rather than a firewall feature.
ALG & Proxy. ALG = application-layer gateway that rewrites embedded addresses/ports so the protocol survives NAT/inspection. Yes = native/managed support. Partial = available via an underlying conntrack/session helper, add-on, or limited scope. No = not supported. N/A = the platform performs no NAT, so NAT ALGs do not apply. Proxy columns: Yes = a built-in proxy of that type; Partial = plugin/limited; No = none.
Operational Mgmt. Yes = native first-party capability (vendor central manager, documented REST API, official IaC provider, native config rollback / commit model). Partial = community/third-party, limited, or indirect. No = none.
HA & Clustering. Yes = native HA/clustering feature. Partial = achievable by pairing with external tooling (keepalived, conntrackd, pacemaker) or limited scope. No = none.
Observability. Yes = native export/telemetry. Partial = via add-on package or limited. No = none. SNMP is generally N/A on managed cloud services.
NGFW Security. IDS and IPS use a delivery scale - Built-in (native; may be subscription-gated for signatures) / Add-on (separate module, container, or premium tier you must enable) / None. App-ID, TLS/deep inspection, URL filtering and threat feeds use Yes/Partial/No, where Partial = subscription tier or limited depth (e.g. FQDN filtering rather than full app-ID).

The two "disabled rule" columns

Count retained when disabled = the accumulated hit count survives disabling the rule (history preserved). Disabled-rule monitor (would-match) = a disabled rule still tallies matches without acting. The second is genuinely rare: on most platforms the practical equivalent is an enabled count-only or log-only rule, which is why that column skews toward No / Partial.

Tier definitions

  • OSS Firewall - dedicated open-source firewall distributions (pfSense CE, OPNsense, IPFire, Endian).
  • Router-Firewall - platforms whose primary identity is routing but that perform stateful firewalling (OpenWrt, VyOS, MikroTik RouterOS, Ubiquiti EdgeOS, Cisco IOS-XE ZBF).
  • Installable Package - Linux/Unix packet-filter engines and front-ends installed on a host OS (iptables, nftables, firewalld, ufw, Shorewall, OpenBSD pf, FreeBSD ipfw, ConfigServer CSF).
  • Commercial NGFW - dedicated commercial security appliances (Palo Alto, FortiGate, Check Point, Cisco FTD, Cisco ASA, Juniper SRX, Sophos, SonicWall, WatchGuard, Barracuda, Forcepoint).
  • Carrier-grade CGN - platforms built for large-scale / carrier NAT, anchoring the NAT-maturity end (A10 Thunder CGN, F5 BIG-IP CGNAT, Juniper MX).
  • Cloud-native - managed cloud firewall services (AWS Network Firewall, Azure Firewall, GCP Cloud NGFW).
